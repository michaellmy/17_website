Project Description

Project Title: Hospital Surveys


Project Abstract

We are designing an online survey tool for hospitals. The surveys are designed by hospital staff using a web application and delivered to patients using a progressive web app. A database stores the anonymised patient feedback, which can be exported for further analysis.

Key Features

  • Progressive web app (works on web and mobile).
  • App allows to design, create, and manage questionnaires stored in a database.
  • App allows child patients to answer questionnaires.
  • App allows GOSH staff to analyse data from questionnaires with admin rights.
  • App is hosted on Heroku.

User Requirements for Hospital Surveys

Gathering User Requirements:


We’ve decided to carry out a semi-structured interview for the two users that would use our product, which are the GOSH DRIVE Admins designing questionnaires, and the child patients answering them.


 

1) GOSH DRIVE Admin Interview:

Q: Should these questionnaires have a fixed format?
A: No, it can be adjusted and customized, like the background colours and images could be changed as they wished.

Q: How do you think the data and answers from the questions should be stored?
A: A database could be used to store the answers from the questionnaires, as well as the login information of users designing the questionnaires.

Q: Should the web app keep track of the child patients answering the questionnaires?
A: It shouldn’t, instead, it could be anonymous to allow the kids to express themselves.

Q: How should patients answer these questionnaires?
A: The patients should be able to access a page using mobile phones and tables that allows them to fill in the questionnaire forms.


2) Child Patient Interview:

Q: Why didn’t you like filling in the questionnaire forms?
A: The form is very boring and hard to fill.

Q: Do you like questionnaires to be colourful or plain?
A: I want them to be colourful!

Q: Would you like if we put some cute animals in the questionnaire?
A: Yes, I really like animals.

MoSCoW List of Requirements:

After doing research and multiple discussions, we came up with a list of requirements after agreements with our project partners.
    • Designed as a Progressive Web App to allow completion of questionnaires using tablets and phones.
    • Admin panel for GOSH staff to create and manage multiple questionnaires.
    • Page that provides patients access to questionnaires.
    • Database to store answers from questionnaires.
    • Deployed of the web-app on Heroku.

    • Classification of different questionnaires for different age groups and patient types (e.g. inpatients, outpatients)
    • Separate questionnaire designs for different age groups.
    • Uncomplicated UI to allow design of questionnaires of users who do not have a background in computer science.
    • Simple design and UI for patients answering questionnaires.
    • Acceptance of different types of answers for the questionnaire designs e.g. drop-down, MCQ, sliders, text.

    • Different UI designs for different patient age groups answering questionnaires.
    • Statistics to analyze general data about existing questionnaires.
    • Statistics to analyse the answers obtained from questionnaires e.g. trends, patterns, etc.

    • Usage of separate frameworks for mobile and web development.
    • Log-in system for patients answering questionnaires.
    • Multilingual surveys.

Related Technologies:

As our application is a new full-stack application, we decided to choose separate frameworks for front-end development and back-end development. After doing spending time learning about these frameworks, we have narrowed down our choices:

  • Django

    Django is a Python framework which follows the model-template-view pattern and was recommended by our clients. In practice, we found it to have many built-in features such as a front-end administration panel, database mapping, and easy authentication [1]. However, time is required to get started to learn the basic Django boilerplate and structure.

  • Flask

    Flask is a light-weight Python framework that is easier to get started with. However, it offers much less built in functionality than Django, and requires developers to be flexible [1] to make the best design decisions. It is easy to get started in developing a REST API in Flask, but functionalities such as authentication is much harder to be implemented in the future.

  • Vue

    • Open-source JavaScript progressive framework that follows the Model–view–Viewmodel pattern.
    • Heavy emphasis on Single File Components, allowing us to build style and functions within one file.
    • Has a gentle learning curve [2] due to its simple API.
    • Fast and light-weight, and is generally straight-forward.

       

  • React

    • An extremely popular framework, hence resource availability is very high.
    • Requires the use of various other libraries for different functionality.
    • Has a slightly steeper learning curve [2] than Vue, but ‘create-react-app’ package helps us get started easily.
    • Contains service workers [3] that allows users to see results even when offline.

      •  

 

Back-end decision:

We have decided to use Django over Flask despite its steeper initial learning curve. This is because we believe Django lets us focus on the actual questionnaire designs in the long run, as a lot of functionality such as authentication and database management is already built-in. We also think that Flask may introduce many bugs in the future due to our inexperience in back-end development so far.



Front-end decision:

Both frameworks were reasonable to us, but we decided to choose React as our front-end JavaScript framework in the end. Despite having a steeper learning curve, we decided that the ‘create-react-app’ package can help us overcome this problem. The service workers also prove to be an advantage, as offline usage is an important property of a progressive web app.

Related Projects:

Our research on projects similar to Hospital Surveys

  • Survey Monkey

    Survey Monkey is a website designed particularly for admins to design questionnaires. It is written in Python and developed with the .NET framework. It offers a wide range of functionalities, including functionalities to import questionnaires from templates, analyse results, and obtain verified responses for your questionnaires. It is very similar to our project, Hospital Surveys, as it allows for questionnaires that are highly customizable[4]. However, not all of these functionalities will be implemented.

  • Google Forms

    Google forms is also designed for users to design questionnaires, and for users to respond to these questionnaires easily[5] and efficiently. Google Forms allows for admins to design questionnaires to obtain responses in a short amount of time. In our project, we plan to implement the simple UI design for users to design questionnaires, but we require a more detailed way to process the data from the responses obtained, i.e. through a database.


 
 

Conclusion


Google Forms offers less customization and functionality than Survey Monkey but is much more user-friendly. However, Survey Monkey is more focused on the questionnaires itself, and places more emphasis on the users creating the questionnaires. They both have their own merits. Google Forms' straight-forward UI and Survey Monkey's customizability were both great features to have for our web-app.

How to setup the Web App

User manual:

https://djreact-surveys.herokuapp.com/

To access the Django back-end Admin, go to
https://djreact-surveys.herokuapp.com/admin/
Admin username: admin
Admin password: admin


Deployment manual:


React Front-End:
To run: Install npm, then change directory to /hospital-surveys. Then run the command: ‘npm start’.
Django Back-End:
To run: Run the command ‘python manage.py runserver’.
Deployment on Heroku:
To deploy on heroku, 'git pull' from the heroku project name, then 'git add.', then 'git push heroku master'
